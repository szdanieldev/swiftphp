SwiftPHP

SwiftPHP Documentation

SwiftPHP is a lightweight and modern MVC framework built with pure PHP. It focuses on simplicity, performance, and developer control without unnecessary complexity.

SwiftPHP provides a clean foundation for building web applications with built-in routing, authentication, localization, and configuration handling.

Installation

1. Clone the Repository

git clone https://github.com/szdanieldev/swiftphp.git
cd swiftphp

2. Create Configuration File

cp config/config.php.example config/config.php

Edit:

config/config.php

Configure your database credentials and application settings.

3. Configure Web Server

Set your document root to:

/public

SwiftPHP does not require Node.js.

app/

Main application logic.

  • Controllers: Handle incoming requests
  • Models: Database and business logic
  • Views: HTML rendering

core/

Framework engine (Router, Request, Response, Auth, Localization loader).

config/

Application configuration files.

localization/

Language files grouped by locale.

public/

Entry point (index.php) and public assets.

routes/

Route definitions.

storage/

Logs and session data.

Basic Usage

Creating a Route

In routes/web.php:

$router->get('/hello', 'HomeController@hello');

Creating a Controller Method

In app/Controllers/HomeController.php:

public function hello()
{
    return view('hello');
}

Creating a View

Create app/Views/hello.php:

<h1>Hello SwiftPHP!</h1>

Visit:

http://localhost/hello

Controllers

Controllers handle HTTP requests and return responses.

Example:

class HomeController extends Controller
{
    public function index()
    {
        return view('home');
    }
}

Controllers:

  • Receive request data
  • Validate input
  • Call models
  • Return views or redirects

Models

Models handle database interaction and business logic.

Example:

class User extends Model
{
    protected $table = 'users';
}

Models are responsible for:

  • Fetching data
  • Saving data
  • Updating records
  • Deleting records

Views

Views render HTML output.

Example:

<h1><?= $title ?></h1>

Views should:

  • Contain minimal logic
  • Focus on presentation
  • Display data passed from controllers

Authentication

SwiftPHP includes a built-in authentication system.

Available Routes

/auth/login
/auth/register
/auth/logout

Features:

  • User registration
  • Login
  • Session handling
  • Logout

Authentication logic can be extended or customized.

Request Lifecycle

  1. Request enters through public/index.php
  2. Core system initializes
  3. Router matches route
  4. Controller executes
  5. View renders output
  6. Response is sent to browser

Each request runs independently and completes its own lifecycle.
